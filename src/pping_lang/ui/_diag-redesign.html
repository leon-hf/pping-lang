诊断 tab · 样式备选 mock 数据示意 ① Feed 卡② 瓶颈分组③ 密集表④ 主从
四套布局都贯彻产品内核:事实(规则名)→ 署名推断 → 处方 三层分明,证据(metric 实测值)可见,瓶颈用 4 regime(A 喂不饱 / B 带宽墙 / C 算力墙 / D 容量墙)上色。挑一套,或说"①的卡 + ②的分组"我再合。
1

证据 Feed(精炼卡)

竖向卡片流,左侧严重度色条;一眼读到「事实 → 推断 → 处方」三段 + 证据 chips。最贴近现状但更克制、更有层次。

适合:默认视图 · 信息密度适中 · 改动最小
2

按瓶颈分组(regime 泳道)

把诊断按 4 个瓶颈处境分列,每列一个 roofline 小图标 + 计数。一眼看出「系统现在卡在哪类墙上」—— 这是只有本产品讲得出的心智模型,适合做卖点。

适合:讲故事 / 教学 / 突出 4-regime 模型
3

密集表(ops 风)

一行一条:严重度 · 规则 · 事实/推断 · regime · 处方。信息密度最高,像 SRE 的告警面板,适合规则多、要快速扫的场景。点行展开看全文。

适合:高密度 · 规则多 · 运维视角
4

主从(列表 + 详情)

左列扫一眼全部命中,右侧看选中那条的完整推理链:事实 → 署名推断 → 证据 → 处方 → 触发条件。适合要给每条诊断「亮全证据」的深读场景。

适合:深读 · 证据×推理完整呈现 · 规则少而精
